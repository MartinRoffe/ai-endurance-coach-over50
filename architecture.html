06 — Coach Chat

Coach Chat — Detailed Architecture

The most complex feature in the app: real-time SSE streaming, a multi-turn tool-calling loop (up to 6 turns), structured retrieval (RAG), a durable coaching memo, and a proposal card that can surgically push a single changed workout to Garmin Connect.

Tool definitions available to Claude

✏️ propose_plan_change

  • Suggest a session duration/type change
  • Parameters: date, duration_min, reason, session_type?, new_label?
  • Frontend renders a confirmation card
  • On approval → POST /apply-plan-change → Garmin push
  • Requires: date, duration_min, reason

🍽️ get_meal_cycle

  • Full 4-week nutrition cycle (every meal + macros)
  • Per-week shopping tally
  • Source: nutrition_plan.meal_cycle_full()
  • No parameters required

🚴 get_activity_analysis

  • Full post-workout AI coaching text
  • Interval rep HR data if present
  • Identify by date (YYYY-MM-DD) or activity_id
  • Defaults to most recent analysed session

😴 get_sleep_history

  • Nightly table: score, duration, stage %, HRV
  • Parameter: days (default 30)
  • Source: history.sleep_history()

📈 get_performance_detail

  • Durability (late-ride HR drift) last 8 rides
  • Foster monotony/strain last 8 weeks
  • Estimated FTP / W/kg history
  • Weekly zone distribution (8 weeks)

🏔️ get_hr_plan

  • Full 46-week Haute Route 2027 plan
  • Week-by-week sessions across 5 phases
  • 7 event stages (km, elevation, key climb)

✅ get_compliance_detail

  • Full per-week plan vs actual breakdown
  • Sessions completed, volume %, status
  • Source: _plan_completion_stats()

📋 get_workout_description

  • 2-sentence coaching note per session label
  • Parameter: label (e.g. "Sweetspot Ride")
  • Source: workout_descriptions cache table

⚡ get_fuelling_plan

  • In-ride carb/fluid/sodium targets
  • Parameters: session_type, dur_min
  • Only for endurance sessions ≥75 min
  • Source: fuelling_plans cache table

🏁 get_event_plans

  • Ghent→Amsterdam Day 1 + Day 2 pacing/fuelling
  • Haute Route Alpes all 7 stage plans
  • Generated by Sonnet, cached in text_cache
  • No parameters required

SSE event types emitted by _stream_coach_sse() and consumed by the browser's ReadableStream reader:
text chunk of streaming reply text — appended live to the bubble
tool read-tool call in progress — shows "🔍 checking your sleep history…"
proposal plan change proposal — renders the blue confirmation card
done stream complete — finalises the bubble (removes streaming cursor)
error exception caught — shows "Something went wrong. Please try again."
